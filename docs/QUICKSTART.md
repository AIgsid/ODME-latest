# ODME Quick Start

This guide is for first-time users who want to launch ODME, open an example, generate scenarios, and find the resulting files.

## 1. Choose How To Run ODME

### Option A: Use a packaged app

If the repository has a tagged release, download the native package for your platform from GitHub Releases:

- Windows: packaged `ODME.exe`
- Linux: app-image archive plus `.deb` and `.rpm`
- macOS: `ODME.app` zip and `.dmg`

This is the simplest option for new users.

### Option B: Build from source

Requirements:

- Java 17 or newer
- Maven 3.8 or newer

Build and run:

```bash
git clone https://github.com/AIgsid/ODME-latest.git
cd ODME-latest
mvn clean package -DskipTests
java -jar target/SESEditor-2.0.0-SNAPSHOT.jar
```

## 2. Open an Example Project

ODME includes two useful starter examples:

- `examples/RunwaySignClassifier`
  Good for learning the SES structure and variable modeling on a smaller project.
- `examples/ScenAIroTableII`
  Good for scenario pruning, automatic scenario generation, constrained sampling, and scenario exports.

To use an example, copy it out of `examples/` so you can edit it freely, then open it in ODME with `File -> Open`.

Recommended first project:

```text
examples/ScenAIroTableII
```

## 3. Understand The Main Modes

### Domain Modeling

Use this mode to define the domain model:

- SES tree and graph structure
- entity variables and ranges
- intra- and inter-entity constraints
- probability distributions
- ontology/schema views

### Scenario Modelling

Use this mode to derive and inspect scenarios:

- prune the SES manually into a PES
- open `Scenario Manager`
- generate multiple scenarios automatically
- review generated scenario folders and exports

## 4. Generate Scenarios Automatically

Using `ScenAIroTableII`:

1. Open the project in ODME.
2. Switch to `Scenario Modelling`.
3. Open `Scenario Manager -> Scenarios List`.
4. Click `Automatic Scenario Generation`.
5. Enter:
   - a scenario name prefix
   - the number of variable samples per structural combination
6. Confirm generation.

ODME will:

- enumerate valid pruning combinations
- apply constrained Latin Hypercube Sampling
- create one scenario folder per generated scenario
- update `scenarios.json`
- refresh the Scenario List
- regenerate export files

## 5. Open And Inspect Generated Scenarios

From `Scenario Manager -> Scenarios List` you can:

- open a scenario in the scenario editor
- refresh the list
- delete selected scenarios
- delete all generated scenarios while keeping `InitScenario`

When you open a scenario:

- the graph view shows the pruned scenario
- clicking a node such as `FlightParameters` shows sampled values in the variable table
- the XML tab shows the generated scenario XML

## 6. Know Where ODME Writes Files

Inside a scenario folder such as:

```text
ScenAIroTableII/AutoScenario_001/
```

you will typically find:

- `ScenAIroTableII.xml`
  Raw structural scenario tree
- `ScenAIroTableII.ssdvar`
  Sampled variable values used by ODME
- `ScenAIroTableII.ssdcon`
  Constraints for that scenario
- `ScenAIroTableII.ssdbeh`
  Behavior data
- `ScenAIroTableII.ssdflag`
  Flags/state metadata
- `xmlforxsd.xml`
  XML shown in the ODME XML pane

Project-level scenario exports are written to:

```text
ScenAIroTableII/ScenarioExports/
```

That folder contains:

- `xml/<ScenarioName>.xml`
  One readable XML copy per scenario
- `scenario-summary.csv`
  One CSV containing selected nodes and sampled variable values across all scenarios in the list

## 7. Recommended First Workflow

For a first pass through the tool:

1. Open `examples/ScenAIroTableII`.
2. Inspect the SES in `Domain Modeling`.
3. Switch to `Scenario Modelling`.
4. Open `Scenario Manager -> Scenarios List`.
5. Generate a small batch first, for example `2` to `5` samples per combination.
6. Open a few generated scenarios and inspect variables, XML, and exports.
7. Use `Delete Generated` if you want to reset and regenerate.

## 8. Common Issues

### The packaged app does not exist yet

GitHub Actions builds native packages, but release assets appear only after a tagged release such as `v2.0.0`.

### Help -> Manual does not open

Use a current build. The manual is bundled with the app and should open from the Help menu.

### A scenario is listed but does not open

Refresh the Scenario List and reopen it. If the scenario was generated by an older build, regenerate it with the current version.

### The XML pane is blank for an older generated scenario

Open it once with the current version or regenerate the scenarios. Current generations write `xmlforxsd.xml` into each scenario folder.

### I created a folder manually and it is not listed

The Scenario List is driven by `scenarios.json`. Use ODME scenario actions so the list stays in sync.
